# cluSTAR Design System

## 📌 프로젝트 개요

**cluSTAR**는 사용자의 생각을 빠르게 기록하고,  
아웃풋으로 연결하는 **아웃풋 중심 메모 플랫폼**입니다.

본 디자인 시스템은 cluSTAR 전반에 걸쳐  
**일관된 사용자 경험(UI/UX)** 과  
**예측 가능한 컴포넌트 사용성**을 제공하기 위해 구축되었습니다.

Storybook을 기반으로 컴포넌트의  
구조, 사용법, 상태를 명확히 정의하고 공유합니다.

---

## 🎯 디자인 시스템의 목표

- UI/UX 일관성 유지
- 컴포넌트 재사용성 극대화
- 협업 시 커뮤니케이션 비용 감소
- 빠른 기능 개발을 위한 안정적인 UI 기반 제공

---

## 🔗 주요 링크

- [GitHub Repository](https://github.com/TEAM-CLUSTAR/CLUSTAR-CLIENT)
- Storybook: 본 문서는 cluSTAR 디자인 시스템의 공식 문서입니다

## 📦 구성 요소 (Components)

디자인 시스템은 다음과 같은 컴포넌트들로 구성됩니다.
(예시)
- **Button**
  - Primary / Secondary
  - Icon Button
  - Toggle Button
- **Input**
- **Modal**
- **Typography**
- **Layout / Spacing**

> 각 컴포넌트는 상태, 인터랙션, 사용 예시를 포함한  
> Story를 반드시 제공합니다.

---

## 🎨 Design Tokens

디자인 토큰은 서비스 전반에서 **단일한 기준**으로 사용됩니다.

- **Color**
  - Primary / Secondary
  - Semantic Color (Success, Error, Warning)
- **Typography**
  - Font Family
  - Font Weight
  - Line Height
- **Spacing**
- **Border Radius**

토큰 변경 시 전체 UI에 일관되게 반영될 수 있도록 관리합니다.

---

## 🧑🏻‍💻 협업 규칙

- 모든 공용 컴포넌트는 **Storybook 스토리 작성 필수**
- 컴포넌트 변경 시 Docs 및 Story 업데이트 필수
- 신규 컴포넌트 추가 시 디자인 토큰 우선 사용
- 접근성(a11y)을 고려한 컴포넌트 설계 권장

---

## 🚀 앞으로의 확장 방향

- Compound Component 패턴 도입
- 접근성 검사 자동화 (a11y addon 활용)
- 디자인 토큰과 코드의 완전한 동기화
- 테스트 가능한 UI 컴포넌트 환경 구축

cluSTAR Design System은  
**함께 만드는 시스템**이며, 지속적으로 개선됩니다.
